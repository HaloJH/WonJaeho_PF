AS-IS JSP 기반 레거시 시스템을 Vue 3 + Spring Boot로 전면 마이그레이션. 요구 분석·화면 설계부터 UI 구현, 백엔드·DB, QA까지 전 영역을 담당했고, 기존 화면을 그대로 옮기지 않고 정보구조·동선을 다시 설계해 재구성. 공용 컴포넌트·SQL 패턴·백엔드 패키지 구조까지 표준을 정립.

01 화면 전면 재구성 · 공용 컴포넌트 시스템 설계

  • AS-IS JSP 화면 제로베이스 재구성 — 단순 이관이 아니라 정보구조·사용자 동선을 다시 설계해 Vue 3로 구현. 시안 의존 없이 화면 설계부터 구현까지 자력 수행.
  • 반복 UI 패턴의 공용 컴포넌트화 — 첨부파일, 폼 입력(날짜·숫자·코드 검색), 이력/버전 선택 등 화면마다 중복되던 UI를 유형별 표준 컴포넌트로 추출.
  • 공통 레이아웃·스타일 표준화 — 테이블·탭·패널 등 공통 요소를 글로벌 스타일 규칙으로 분리해, 신규 화면이 별도 작업 없이 디자인 가이드를 그대로 따르도록 설계.
  • 조합형 개발 체계 확립 — 신규 화면은 표준 컴포넌트 조합만으로 구현 → 코드 중복 제거, 화면 간 UI 일관성 확보, 유지보수성 향상.

02 백엔드 구조 리팩토링

  • 같은 특성 서비스끼리 패키지 분리 → 프론트 폴더구조와 백엔드 패키지 일치시켜 유지보수성 향상.
  • AS-IS의 복잡한 서비스 구조 개선 — 각 서비스 별로 흩어져 있던 Service 및 JSP를 하나로 통합해 가독성·디버깅 개선.
  • BIZ_MASTER / BIZ_PORTAL / NBM_BIZ_MASTER 등 핵심 테이블의 proStep 스코프 격리 + 역할코드 정리.

03 AS-IS → TO-BE 매핑 및 검증

  • 매퍼 grep 분석으로 실제 사용 테이블 추출 → DSN 산출물(테이블 목록/정의서) 정합성 검증 자동화.
  • 24건 잉여 정의서 식별 + 6건 누락 정의서 추가. 3종 매트릭스(DSN_09 ↔ DSN_10 ↔ 매퍼) 비교 스크립트 작성.
  • AS-IS 마이그레이션 정책 준수 — 위험 발견 시 임의 개선 대신 협의 항목으로 분리.

주요 성과

  • ✓ 반복 UI를 공용 컴포넌트로 표준화 → 신규 화면 개발 시간 단축
  • ✓ 백엔드 서비스 구조 통합으로 가독성·디버깅 개선
  • ✓ DSN 산출물 매트릭스 검증 자동화로 휴먼 에러 사전 차단

06 AI 활용 역량

AI를 단순 코드 생성 도구가 아닌, 코드베이스를 함께 분석하고 설계를 검토하는 페어 프로그래밍 파트너로 활용합니다. KOPMS 프로젝트에서 Claude Code를 적극 도입하여 실제 생산성을 끌어올렸습니다.

활용 철학

도구가 아닌 협업자로 본다

단순히 "코드 짜줘"가 아니라, 함께 코드베이스를 읽고 패턴을 발견하고 의사결정을 검토하는 동료처럼 활용합니다.

최종 판단은 사람이 한다

AI 제안을 그대로 받아들이지 않습니다. 메모리/룰 기반으로 프로젝트 컨벤션을 명시하고, AS-IS 마이그레이션 정책 등은 임의 개선 금지로 가드합니다.

맥락 관리에 신경 쓴다

프로젝트 메모리 파일을 적극 활용해 컴포넌트 사용법·DB 스키마·코드 컨벤션을 영속화하여 매 세션마다 일관된 협업 환경을 유지합니다.

실전 활용 사례 — KOPMS 프로젝트

CASE 01

DSN 산출물 정합성 자동 검증

문제 AS-IS 산출물(테이블 목록·정의서)이 실제 사용 테이블과 일치하는지 수기 검증 부담.
접근 MyBatis 매퍼 XML을 grep으로 파싱 → 실제 사용 테이블 추출 → DSN_09 / DSN_10 / 매퍼 3종 매트릭스 비교 스크립트 작성.
결과 잉여 정의서 24건·누락 정의서 6건 자동 식별. 휴먼 에러 차단 + 산출물 신뢰도 향상.
CASE 02

공용 컴포넌트 패턴 합의 및 표준화

문제 신규 화면 추가 시 다이얼로그·테이블·점수판 등이 화면별로 제각각.
접근 기존 화면 코드 분석으로 공통 패턴 추출 → BrFileAttach·br-table-fill·score-panel 등 글로벌 utility로 추출. 메모리에 사용 가이드 영속화.
결과 신규 화면 개발 시 컴포넌트 import만으로 디자인 가이드 자동 반영. 코드 재사용성 향상.
CASE 03

백엔드 패턴 일관성 가드

문제 MyBatis OGNL 분기·jdbcType 누락·string compare 등 미세한 컨벤션 차이로 인한 버그 발생.
접근 발견한 컨벤션을 메모리에 명시 — "OGNL 큰따옴표 패턴", "BrNumberField → jdbcType=VARCHAR", "CASE WHEN 우선" 등.
결과 같은 패턴 실수 재발 방지. 코드 리뷰 시간 단축. 신규 화면도 동일 컨벤션 자동 준수.
CASE 04

AS-IS 마이그레이션 정책 자동 가드

문제 AS-IS 코드 마이그레이션 시 위험 발견하면 본능적으로 개선하고 싶은 충동 → 고객사 정책 위반.
접근 "AS-IS 마이그레이션 코드 임의 개선 금지" 룰을 메모리에 명시. 위험 발견 시 자동으로 협의 항목으로 분리 제안.
결과 고객사 정책 100% 준수 + 위험 항목 별도 트래킹으로 운영 안정성 확보.

이러한 활용 방식은 단순 코드 생성 보조를 넘어, 시스템 전반의 일관성을 유지하는 메타 분석 도구로 AI를 활용하는 방법입니다. 앞으로도 AI 도구의 발전을 적극 학습하며, 더 큰 시스템에서 더 일관된 결과물을 만들어내는 개발자가 되고자 합니다.

07 마무리

가치관

맥락 위에서 결정한다

문제를 발견하면 즉시 고치고 싶은 충동이 들지만, 고객사 정책·운영 영향·기존 컨벤션 등 맥락을 먼저 확인합니다. "왜 이렇게 되어 있는지"를 이해한 후에 손을 댑니다.

재사용 가능한 형태로 정리한다

한 번 풀린 문제는 다음에 같은 문제가 와도 풀릴 수 있도록 패턴화합니다. 공용 컴포넌트·utility class·메모리 룰 등은 모두 이 원칙의 산물입니다.

책임지는 코드를 쓴다

내가 작성한 코드가 운영에서 어떻게 동작할지 끝까지 추적합니다. 단순 화면 구현이 아닌, 데이터가 어떻게 흘러 어디에 저장되는지 이해하고 코드를 작성합니다.

앞으로 학습 방향

  • 인프라/배포 영역 학습 — Linux 명령·로그 분석·CI/CD 파이프라인
  • Spring Security / SSO(SAML, OAuth 2.0, OIDC) 인증 구조 심화
  • Docker / Kubernetes 기반 컨테이너 배포 경험
  • AI 활용 사례 지속 발굴 및 팀 내 공유
